Takeaways
• Vector search returns chunks, not files.
The agent gets fragments without surrounding context. In django-11910, the vector agent made
26 searches and still couldn't piece together how three functions interact across files — the
agentic agent just read them.
• Agentic search enables chain-of-thought exploration.
In pr-2532 (ts-go), the agentic agent worked like a developer: grepped for the function name,
read the full function, spotted a flaw in the cycle detection logic, then followed callers to confirm
its theory before applying a targeted fix. The vector agent couldn't do this — it just kept querying
the database with different keywords, never building on what it already found.
• More searches ≠ better results.
Across ts-go tasks, vector used 2.8x more LLM calls, 3.1x more tokens, and cost 2.8x more per
task — while performing the same or worse.
